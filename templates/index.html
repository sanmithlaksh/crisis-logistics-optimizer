Demonstration and Presentation Guide

DAA Algorithm Explanations

Use this guide to walk your teammates or evaluators through a live demonstration of the crisis logistics system. Below are step-by-step instructions for each algorithmic component.

DEMO 1: Max-Heap Priority Ranking

Goal: Show how the system ranks incoming requests by urgency using a Binary Max-Heap.

Steps:

  1. Go to the Requests & Heap tab.
  2. In the Generate Crisis Relief Request form, submit two custom requests:
    • Request A: Zone E (Santa Monica) (Severity: Low), Resource: Medicine, Quantity: 50.
    • Request B: Zone A (Glendale) (Severity: Critical), Resource: Water Crate, Quantity: 200.
  3. Observe the Binary Max-Heap Tree on the right.

Explanation: Request B automatically bubbles up to the root node (index 0) due to Glendale's high priority. Score formula: Priority = (Severity Weight * 10) + (Population / 10,000). Heap maintains sorting in O(log n) time.

DEMO 2: Greedy Resource Allocation

Goal: Show how supplies are distributed to the most critical zones first, handling inventory depletion.

Steps:

  1. Go to the Requests & Heap tab.
  2. Click the green Allocate Resources button.
  3. Watch the Status column change to "Allocated", "Partial", or "Unfulfilled".
  4. Go back to the Resource Inventory tab and check the stock levels.

Explanation: A greedy strategy pops the highest-priority request off the heap and allocates remaining warehouse stock up to min(Requested, Available), maximizing immediate disaster mitigation.

DEMO 3: Risk-Weighted Dijkstra Routing

Goal: Show how the system finds the safest path by avoiding active hazards.

Steps:

  1. Go to the Route Visualizer tab.
  2. Click on Zone H (Malibu) or Zone J (Anaheim) on the map.
  3. Click Inspect Route in the popup.
  4. Watch the cyan line animation detour around red (Blocked) or blue (Flooded) roads.

Explanation: Path selection is based on Effective Weight = Distance * Hazard Multiplier. Flooded roads increase the cost by 3.0x, while blocked roads cost 9999.0x, forcing the route around hazards.

DEMO 4: 0/1 Knapsack Cargo Loading

Goal: Show how vehicles optimize cargo weight limits to carry the most valuable items.

Steps:

  1. Select a scenario in the header (e.g. Flood Emergency) and click Run Sim.
  2. Go to the Fleet Assignment tab.
  3. Scroll down to the Knapsack Vehicle Cargo Load Optimizations panel.

Explanation: Evaluates items to fit vehicle weight capacity while maximizing total priority utility. Solves the Dynamic Programming formula: DP[i][w] = max(DP[i-1][w], DP[i-1][w-w_i] + v_i).

DEMO 5: Branch & Bound & Comparison

Goal: Show how the fleet is optimally matched to zones and compare results to FCFS.

Steps:

  1. Look at the Branch and Bound Decision Logic on the Fleet Assignment tab.
  2. Navigate to the Sim & Analytics tab to compare FCFS with the AI Engine.

Explanation: Branch & Bound matches vehicles to zones to minimize Cost = Travel Cost + Route Risk + Unfulfilled Cargo Penalty, pruning sub-optimal branches. The analytics charts show our AI Engine outperforms FCFS on all metrics.

ADVANCED: Real-Time Dynamic & Fallback Demos

Dynamic Recalculation: Inspect the route to Burbank under "Flood Emergency" versus "Conflict Zone". The route changes dynamically as hazard profiles shift.

Database Resilience: If Supabase is offline, the app switches to the local SQLite database fallback (badge changes from green "Supabase Mode" to yellow "SQLite Mode") without interrupting the system.
